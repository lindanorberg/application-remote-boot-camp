Mob programming

Mob programming is writing code togheter in a team. You have one computer (with a TV screen preferably so everyone can see). There are two key jobs, the driver writing the code on the computer and the navigator who tells the driver what to write. In the team you take turns (every 10-15 minutes) on having the different roles. A team consists of 4-5 persons. The rest of the team does not relax in the back seat, they solve the code problems togheter along the way. The pros of mobprogramming is that the code you submit after a coding session is more correct and problems are already solved. The cons may be that it does not suit everybody, some think better silently and alone.
